# DESIGN_PRINCIPLES.md - AI Workflow Governance Designer

## 1. Governance First
Design is secondary to the structure and integrity of the governance record. Every UI element should serve the purpose of clarifying accountability and risk.

## 2. Professionalism and Credibility
The application must feel like a high-end engineering tool. Use a refined, professional aesthetic (dark mode, clean typography, subtle animations) to convey authority and precision.

## 3. Explicit Boundaries
The "Human-AI Boundary" is the core concept. The UI must clearly differentiate between human actions, AI support, and automated AI generation.

## 4. Capability-Driven Development (CDD)
Every feature must map back to a specific capability intent (Traceability, Accountability, Risk Awareness, Reviewability).

## 5. Local-First & Privacy-Preserving
The user is the sole owner of their data. No data should ever leave the browser unless explicitly exported by the user.

## 6. Inspectability over Abstraction
Avoid "magic" or hidden logic. The user should be able to see and inspect how the Governance Completeness Index is calculated.
